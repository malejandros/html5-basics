Introduction

HTML5 stands for HyperText Markup Language. It is used to create content on the world wide web like webpages. HTML5 is the fifth revision of the language's standard. It was published in October 2014, 17 years after HTML4 was standardized.

HTML5 aims to provide improved support for latest multimedia. It also reduced dependency on plugins and APIs providing a common interface making loading elements easier...

API

API stands for Application Programming Interface. It is a set of routines, protocols, and tools for building software and applications. This content uses the aside element. This is only for semantic meaning and so doesn't carry any instructions for presentation. It will appear like a normal paragraph in your document. If you wish to include this in a sidebar, you will have to provide the appropriate markup and styling for it.

Features

Keeping up with times

HTML5 introduced elements and attributes that fits the typical usage in modern websites. It introduced new semantic elements to replace generic blocks like div and span. Purely presentation elements such as font and center have also been completely dropped...

New APIs

New APIs were introduced to allow cross-document communication, drag and drop functionality, immediate mode 2D drawing using the canvas element, webstorage etc...

Error Handling

HTML5 provides detailed rules for parsing allowing HTML5 compliant browsers to produce the same type of behavior when parsing incorrect syntax. It has been designed for compatibility with older browsers. Older browsers ignore new HTML5 constructs...
